# REST API v1

Headless/external API for programmatic access. All endpoints use Bearer token authentication via the organisation's API key.

**Base path:** `/api/v1`  
**Auth header:** `Authorization: Bearer rmai_<key>`

The API key is validated by `src/middleware/api-auth.ts` which looks up the key in `Organisation.apiKey`. All queries are automatically scoped to the authenticated organisation.

---

## Authentication

```bash
curl https://your-domain.com/api/v1/projects \
  -H "Authorization: Bearer rmai_Su1Bs8Q_1HTL0MsUUhy2-MsVRidCahcM"
```

All endpoints return `401` if the key is missing or invalid.

## POST /api/v1/ai/chat

Natural language interface to Guardian AI. Claude has access to 10 tools that can read and write project data.

**Request:**
```json
{
  "message":   "What are my at-risk projects?",
  "projectId": "optional-project-id",
  "sessionId": "optional-session-id"
}
```

**Response:**
```json
{
  "message":          "You have 2 at-risk projects: Alpha (SPI 0.71) and Beta (SPI 0.83)...",
  "actionsPerformed": [
    { "tool": "get_projects", "summary": "get_projects executed" }
  ],
  "ok": true
}
```

### Available Tools

| Tool | Description |
|------|-------------|
| `get_projects` | List projects with health scores, optionally filtered by status |
| `get_project_detail` | Full project detail including sprints, risks, team |
| `create_risk` | Add a risk to a project |
| `update_feature_status` | Change a feature/task status |
| `create_feature` | Add a feature to a sprint |
| `update_project_status` | Change project status (ACTIVE/PAUSED/COMPLETED/ARCHIVED) |
| `get_alerts` | Get active unresolved alerts |
| `run_guardian_analysis` | Trigger Guardian background analysis |
| `update_ui_config` | Update org UI settings (colors, theme, language) |

The chat loop runs up to 5 iterations (tool call → result → next call) before returning the final text response.

## Shared EVM Library

All v1 endpoints use `src/app/api/v1/_lib.ts`:

- **`computeEvm(p: ProjectRow)`** — Calls `calculateHealth()` with data assembled from a Prisma query result. Returns `HealthReport` extended with `costActual`, `costEstimated`, `openRisksCount`.
- **`DB_PROJECT_INCLUDE`** — Shared Prisma include object used by all v1 project queries (phases, sprints+features, risks, assignments+resource, guardianReport).
- **`phaseRange(index, total, start, end)`** — Computes proportional start/end dates and time-based progress for a phase given its index in the overall project timeline.
