# Scenarios — Governance

Governance covers approval workflows, compliance checkpoints, scope change control, and audit requirements. A well-governed project has documented decisions, approved artefacts, and a clear audit trail.

---

## Governance Artefacts in RoadmapAI

| Artefact | Model | States | Who Approves |
|----------|-------|--------|--------------|
| Functional Analysis (FA) | `FunctionalAnalysis` | `DRAFT` → `PENDING_APPROVAL` → `APPROVED` / `REJECTED` / `REVISION_REQUESTED` | CEO, ADMIN, PMO |
| Project Document | `ProjectDocument` | `DRAFT` → `UNDER_REVIEW` → `APPROVED` | CEO, ADMIN, PMO |
| Scope Change | implicit (project PATCH + statusLog) | Status change logged to `ProjectStatusLog` | PMO, ADMIN |
| Budget Change | implicit (budgetTotal PATCH + snapshot) | Auto-snapshot triggered | PMO, ADMIN |
| Closure Report | `ClosureReport` | Generated on status → CLOSED | Auto-generated by system |

---

## Governance Gaps — What Guardian Checks

Guardian AI flags governance gaps when:

1. **FA not approved** but project is in Sprint 3+ (should have approval before development)
2. **No documents** on a project with > 3 sprints
3. **Scope change** (status log shows ACTIVE → ON_HOLD → ACTIVE) with no note recorded
4. **Budget change** with no snapshot explaining the reason
5. **Project CLOSED** but no closure report generated
6. **High-risk project** (healthScore < 50) with no stakeholder update in 7+ days

## Governance Scenarios

### Scenario A — FA Never Approved
**Context**: Project in Sprint 4. Functional Analysis was created in Sprint 1 but never moved beyond DRAFT.
**Guardian response**:
- Alert category: `governance`, level: warning
- "Functional Analysis for [project] has not been approved. Development in Sprint 4 may be proceeding without confirmed scope. Request immediate approval or acknowledge as informal."
- Action: Route to CEO or PMO for review
- Risk: scope creep, rework after approval

### Scenario B — Document Rejected, No Revision
**Context**: A key design document was REJECTED 10 days ago. No new version submitted.
**Guardian response**:
- "Document '[name]' was rejected [N] days ago with no revision submitted. This is a governance blocker."
- Severity escalates with age: 3 days = warning, 7 days = critical
- Recommend: "Assign revision to [author]. Set deadline."

### Scenario C — Scope Change Without Documentation
**Context**: `ProjectStatusLog` shows status changed from ACTIVE → ON_HOLD → ACTIVE within 48h with no note.
**Guardian response**:
- "Status change sequence detected without a documented reason. This may indicate an undocumented scope decision."
- Recommend: "Add a note to the status log explaining the hold reason and resolution."

### Scenario D — Project Close Without Closure Report
**Context**: Project status changed to CLOSED. No closure report exists.
**Guardian response**:
- This triggers `generateClosureReport()` automatically via the PATCH handler
- If the system failed to generate: "Closure report not found for completed project. Generate manually at Settings > Reports."
- Closure reports contain: final health score, budget variance, lessons learned placeholder, team contributions

### Scenario E — Budget Changed, No Snapshot
**Context**: `budgetTotal` changed via PATCH. Auto-snapshot should have fired.
**Guardian response**:
- If snapshot missing: "Budget change detected without a baseline snapshot. Historical comparison will be incomplete."
- Recommend: "Create a manual snapshot from the project timeline view with note: 'Budget revision: [reason]'."

## Governance Approval Flow (FA Example)

```
PM creates FA → status: DRAFT
  │
  └─ PM submits → status: PENDING_APPROVAL
       │
       ├─ CEO/PMO/ADMIN reviews
       │
       ├─ APPROVED → FA locked, project can proceed
       │
       ├─ REVISION_REQUESTED → note added, FA returns to PM
       │    └─ PM revises → re-submits → PENDING_APPROVAL
       │
       └─ REJECTED → FA must be recreated
```

Only `can.approveDocuments(role)` users can approve/reject/request revision:
- ADMIN: yes
- PMO: yes  
- CEO: yes
- STAKEHOLDER: no
- DEV: no

## Compliance Audit Trail

Every governance action is logged:
- `ProjectStatusLog` — who changed status, when, note
- `FAVersion` — each FA revision with changedBy, version, changeNote
- `ProjectDocument` status field — last updated by whom

Guardian AI can reconstruct the governance timeline from these logs and identify gaps. When asked "was this project properly governed?", Guardian checks all three sources.
